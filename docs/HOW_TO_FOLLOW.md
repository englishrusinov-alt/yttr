# КАК СЛЕДОВАТЬ СИСТЕМЕ — КРАТКАЯ ВЫЖИМКА

## Золотое правило

```
СНАЧАЛА ПОПРОБУЙ → ПОТОМ ПРОВЕРЬ → ПОТОМ ОБЪЯСНИ → ПОТОМ ПОВТОРИ ПОЗЖЕ
```

Если ты делаешь только одно — делай это.

---

## Дневной ритм (3 часа)

| Блок | Время | Что делать |
|---|---|---|
| 🔨 Build | 90 мин | Пиши код по текущему тикету. БЕЗ AI первые 20–30 мин |
| 📖 Docs | 45 мин | Читай доки с конкретным вопросом. Конвертируй в код |
| 🐛 Debug/Test | 30 мин | Пиши тест или чини баг. Классифицируй ошибку |
| 📝 Postmortem | 15 мин | 3 предложения: что сломалось, почему, что делать иначе |

**Если есть только 1 час** → делай только Build.
**Если есть только 30 мин** → делай Recall: восстанови вчерашний паттерн по памяти.

---

## Как работать с тикетами

1. **Бери 1 тикет** — не 3, не 5
2. **Прочитай задачу** — пойми что нужно ДО написания кода
3. **Попробуй сам 20–30 мин** — без AI, без Google, без Stack Overflow
4. **Застрял?** → см. раздел «Тупики» ниже
5. **Работает?** → напиши хотя бы 1 тест + 1 failure case
6. **Готово?** → запиши postmortem (даже 3 предложения)
7. **Следующий тикет** — только после Definition of Done

### Definition of Done (для каждого тикета)
- [ ] Работает на happy path
- [ ] Хотя бы 1 failure path обработан
- [ ] Есть хотя бы 1 тест
- [ ] Логика в правильном слое (router тонкий, логика в service)
- [ ] Logging/error handling не забыт
- [ ] Ты можешь объяснить что сделал и почему

## 🚧 Тупики: что делать когда застрял

### Уровень 1: «Не знаю синтаксис» (5 мин тупика)
→ Погугли конкретный вопрос, прочитай доку, вернись к коду.
→ Это НЕ повод звать AI — это повод учить доку.

### Уровень 2: «Не понимаю как спроектировать» (15 мин тупика)
→ Нарисуй на бумаге: какие данные → куда текут → какие таблицы.
→ Напиши pseudocode без синтаксиса.
→ Если не помогло → спроси AI: *«Как обычно проектируют X? Дай 2 варианта и trade-offs»*

### Уровень 3: «Код не работает и не понимаю почему» (30 мин тупика)
→ Читай traceback ЦЕЛИКОМ, снизу вверх.
→ Добавь print/log на каждом шаге.
→ Упрости: убери всё лишнее, оставь минимальный пример.
→ Если не помогло → спроси AI, но покажи traceback и свою гипотезу.

### Уровень 4: «Вообще не понимаю тему» (весь блок тупик)
→ **Стоп.** Переключись в Docs mode: читай доку 30 мин.
→ Найди worked example (пример в доке), разбери его.
→ Скопируй пример, запусти, поменяй 1 вещь, посмотри что изменилось.
→ Вернись к тикету завтра — сон закрепит.

### Уровень 5: «Не хочу, нет мотивации, лень»
→ Это не тупик. Это усталость.
→ Возьми самый маленький тикет, который можно сделать за 20 мин.
→ Или сделай Recall: восстанови auth flow по памяти.
→ Или просто отдохни — система не сломается за 1 день.

## 🤖 Как использовать AI правильно

### ✅ ИСПОЛЬЗУЙ AI для:
- Code review: *«Проверь этот код на ошибки, security, и design»*
- Объяснения: *«Почему SQLAlchemy делает N+1 тут?»*
- Генерации тестов: *«Какие edge cases я пропустил?»*
- Failure analysis: *«Вот traceback, что сломалось?»*
- Backlog: *«Разбей эту фичу на 3 подтикета»*
- Design critique: *«Вот два варианта, какой лучше и почему?»*

### ❌ НЕ ИСПОЛЬЗУЙ AI для:
- Написания первой версии кода (сначала напиши сам!)
- «Сделай мне auth» — ты ничему не научишься
- Копирования без понимания — если AI дал код, ты ОБЯЗАН:
  1. Прочитать каждую строку
  2. Объяснить зачем она
  3. Переписать ключевые части своими словами

### 🔑 Правило 80/20:
> **80% кода пиши сам. 20% — с помощью AI.**
> Если пропорция обратная — ты учишь AI, а не себя.

## 📅 Еженедельный чеклист

Каждое воскресенье, 15 минут:

- [ ] Сколько тикетов закрыл с честным DoD?
- [ ] Какой тип ошибки встречался чаще всего?
- [ ] Какой паттерн стал автоматическим?
- [ ] Что развалится, если я попробую через неделю по памяти?
- [ ] Где AI помог, а где создал иллюзию знания?
- [ ] Что повторить на следующей неделе?

---

## 📆 Ежемесячный аудит (30 мин)

Честно ответь ДА/НЕТ:

1. Могу написать auth с нуля без AI? [ ]
2. Могу спроектировать 3–5 таблиц с FK? [ ]
3. Могу написать нетривиальный SQL? [ ]
4. Могу объяснить зачем нужны транзакции? [ ]
5. Могу написать осмысленный тест? [ ]
6. Могу дебажить по логам? [ ]
7. Могу объяснить кэширование и инвалидацию? [ ]
8. Могу отличить sync от async? [ ]
9. Могу найти дыру в permissions? [ ]
10. Могу рефакторить без паники? [ ]

**< 5 «Да»** → вернись к базовым фазам.
**5–7** → ты на пути к mid.
**8+** → пора усложнять и добавлять specialization.

---

## ⚡ 5 правил максимальной эффективности

1. **Recall > Restudy** — вспоминать мощнее, чем перечитывать
2. **Один тикет > пять теорий** — делай, а не планируй
3. **Тест + failure > красивый happy path** — баги учат больше фич
4. **Маленький шаг каждый день > марафон раз в неделю**
5. **Объяснил = понял. Не объяснил = иллюзия**

---

## 🛌 Не забывай

- Сон = часть обучения. Мозг консолидирует ночью.
- Прогулка 15 мин между блоками = +20% к фокусу.
- Вода. Серьёзно.
- Если 3 дня подряд нет прогресса — уменьши scope, а не бросай.
